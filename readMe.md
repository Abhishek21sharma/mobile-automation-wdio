Why CSS doesn't exist in MobileWhen you use React Native, you are writing JavaScript, but that code is compiled into native UI views (e.g., UIView on iOS or android.view.View on Android).Web: The browser parses HTML/CSS into a DOM tree. CSS selectors navigate that DOM. Mobile: The app is a tree of native widgets. There is no concept of a "class" or "id" attribute in the CSS sense. The "styles" you write in React Native are actually just layout instructions that the native OS uses to position those native widgets.
Use Predicate Strings (iOS) / UIAutomator (Android): If you need to find an element based on complex logic (e.g., "the button that contains the text 'Submit' AND is a child of the 'Header' view"), you don't use CSS; you use platform-specific predicates. iOS: You can use Predicate Strings (e.g., label == 'Submit' AND name == 'login-button'). This is actually more powerful and precise than CSS.Android: You use UIAutomator selectors (e.g., new UiSelector().text("Submit"))
Summary for your assignment:

1. Don't use CSS. It literally does not exist in the mobile view hierarchy.
